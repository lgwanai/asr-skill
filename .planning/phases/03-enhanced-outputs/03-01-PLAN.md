  <files>asr_skill/postprocessing/formatters.py</files>
  <action>
Add `format_srt_timestamp()` and `format_srt()` functions to formatters.py.

**format_srt_timestamp(ms: int) -> str:**
- Convert milliseconds to SRT timestamp format: HH:MM:SS,mmm (NOTE: comma for milliseconds, not period)
- Example: 3661500 -> "01:01:01,500"

**format_srt(result: dict[str, Any]) -> str:**
- Use sentence_info or sentences from result (same pattern as format_txt)
- Apply detect_overlaps() to segments
- For each segment, output:
  1. Index number (1-indexed)
  2. Timestamp line: "start --> end" using format_srt_timestamp
  3. Text with optional speaker prefix: "[Speaker A] text"
  4. Blank line

**Reference the RESEARCH.md code examples for exact implementation patterns.**

Key details:
- SRT uses COMMA for milliseconds (00:00:00,000), not period
- Include speaker label in brackets when available: "[Speaker A] text"
- Do NOT add [OVERLAP] tag in SRT (subtitle players won't understand it)
</action>

Run: python -c "

assert '[Speaker A]' in output
print('SRT format OK')

- Include Default style and SpeakerA/B/C styles with distinct colors
- Use high-contrast colors: Yellow (SpeakerA), Cyan (SpeakerB), Magenta (SpeakerC)

**format_ass_timestamp(ms: int) -> str:**
- Convert milliseconds to ASS format: H:MM:SS.cc (centiseconds, not milliseconds)
- Example: 3661500 -> "1:01:01.50"

**format_ass(result: dict[str, Any]) -> str:**
- Start with ASS_HEADER
- Use sentence_info or sentences from result
- Apply detect_overlaps() to segments
- For each segment, output dialogue line:
  - Format: "Dialogue: 0,start,end,Style,,0,0,0,,text"
  - Style = "SpeakerA", "SpeakerB", etc. based on spk field (use chr(ord('A') + spk))
  - Escape ASS special characters: backslash, curly braces
  - Do NOT include overlap markers in subtitle text

**Color scheme for speakers:**
- SpeakerA: &H0000FFFF (yellow)
- SpeakerB: &H00FF00FF (cyan)
- SpeakerC: &H00FFFF00 (magenta)
- SpeakerD: &H0000FF00 (green)
- SpeakerE: &H000080FF (orange)
</action>
