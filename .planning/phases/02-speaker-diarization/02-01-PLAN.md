Create `asr_skill/preprocessing/video.py` with FFmpeg-based audio extraction.

Implementation requirements (from CONTEXT.md locked decisions):
1. **Supported formats**: MP4, AVI, MKV only (most common video formats)
2. **Extraction method**: FFmpeg CLI via subprocess module
3. **FFmpeg dependency**: Use imageio-ffmpeg for auto-installation if FFmpeg not found
4. **Temp files**: Auto-delete extracted audio after transcription (context manager pattern)

Module structure:
```python
SUPPORTED_VIDEO_FORMATS = [".mp4", ".avi", ".mkv"]

def is_video_file(file_path: str) -> bool:
    """Check if file is a supported video format."""
    ...

def get_ffmpeg_path() -> str:
    """Get FFmpeg executable path, preferring system FFmpeg, falling back to imageio-ffmpeg."""
    ...

@contextmanager
def extract_audio_from_video(video_path: str):
    """Context manager for video audio extraction with auto-cleanup.

    Yields path to extracted 16kHz mono WAV file.
    Auto-deletes temp file on context exit.
    """
    ...
```

Key implementation details:
- Use `imageio_ffmpeg.get_ffmpeg_exe()` to get bundled FFmpeg binary path
- FFmpeg command: `ffmpeg -i input.mp4 -vn -acodec pcm_s16le -ar 16000 -ac 1 -y output.wav`
- Use `tempfile.mktemp(suffix=".wav")` for temp file path
- Use try/finally in context manager for guaranteed cleanup
- Raise `RuntimeError` with clear message if FFmpeg fails (include stderr in message)

Do NOT:
- Use pydub or moviepy (per research - FFmpeg CLI is more reliable)
- Leave temp files after transcription (use context manager pattern)
- Assume FFmpeg is in PATH (use imageio-ffmpeg as fallback)
