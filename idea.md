# 项目代号：基于FunASR的智能语音解析Skill需求说明书

## 1. 项目概述与目标
本Skill旨在基于阿里开源FunASR框架，集成业界领先的语音识别与后处理模型，打造一个高性能、全功能的本地化语音解析工具。目标用户包括需要处理会议记录、访谈转录、讲座存档及视频字幕生成的内容创作者、研究人员及企业用户。

**核心价值主张**：在保障数据隐私（本地化处理）的前提下，实现对**超长音频/视频**的高精度转写，并通过**说话人分离**技术输出结构化的智能纪要。

## 2. 核心功能需求

### 2.1 超长音频/视频智能转写
- **功能描述**：支持输入时长超过1小时的大文件，系统需具备自动分割、并行处理及结果拼接能力，避免内存溢出。
- **技术指标**：
    - **分段机制**：基于VAD进行智能断句或在静音处切分，默认分段时长建议为300秒（可调），确保语义连贯 。
    - **格式支持**：输入需支持常见音频（MP3、WAV、M4A、FLAC）及视频（MP4、AVI、MKV）格式。针对视频文件，Skill应自动剥离音轨后进行处理 。
    - **采样率标准化**：自动将输入音频统一重采样至16kHz（模型最优采样率），以保证识别精度 。

### 2.2 多说话人分离与角色标记
- **功能描述**：识别音频中的不同声纹特征，在输出结果中标记发言人（如说话人A、说话人B），并精确记录每位发言人的起止时间戳。
- **技术指标**：
    - **实现路径**：采用 **FSMN-VAD（语音活动检测） + Paraformer（语音识别） + CAM++（说话人识别）** 的串联pipeline 。
    - **输出精度**：支持词级别或句级别的时间戳，准确区分顺序发言场景下的说话人 。
    - **重叠说话处理**：对于重叠语音，需在文档中标明重叠时段，避免强制分离导致的文本混乱 。

### 2.3 推理性能优先
- **功能描述**：采用FunASR工具库中精度最高且推理速度最快的模型组合，以满足大规模任务的处理效率。
- **模型选型**：
    - **ASR模型**：选用 **Paraformer-Large**。该模型采用非自回归架构，推理速度较传统Transformer快3-5倍，同时保持较低的字错误率（CER） 。
    - **辅助模型**：启用CT-Transformer标点恢复模型，为裸识别结果添加标点，提升可读性 。

## 3. 非功能需求

### 3.1 输出结果结构化
- **格式要求**：支持多种导出格式以满足不同场景。
    - **TXT**：纯文本全文。
    - **SRT/ASS**：带时间戳的字幕文件，可直接用于视频剪辑 。
    - **JSON**：结构化数据，包含详细的段落信息、说话人标签、词级别时间戳及置信度 。

### 3.2 热词与垂直领域适配
- **功能描述**：允许用户通过配置文件注入热词（如专业术语、品牌名、人名），动态提升特定词汇的识别优先级，优化垂直领域（如医疗、法律、IT）的准确率 。

## 4. 技术架构与部署要求

### 4.1 环境检测与自动化配置
Skill在首次启动时必须具备自我检测与配置能力：

| **检测项** | **要求与动作** |
| :--- | :--- |
| **硬件加速** | 检测NVIDIA GPU及CUDA版本。若有可用GPU，自动加载CUDA加速；若无可自动回退至CPU模式，但需给出性能预警 。 |
| **系统依赖** | 检测 `ffmpeg`。若缺失，自动下载并添加至环境变量，用于多格式音视频解码 。 |
| **Python环境** | 检测Python版本（需≥3.8），并自动创建虚拟环境安装 `funasr`、`torch`、`pydub` 等依赖 。 |

### 4.2 模型资产管理
- **下载机制**：若本地无模型缓存，自动从ModelScope魔搭社区下载指定的模型文件。
- **缓存策略**：模型文件需持久化存储在本地 `models/` 目录，避免重复下载。
- **必选模型清单**：
    - `damo/speech_paraformer-large_asr_nat-zh-cn-16k-common-vocab8404-pytorch` 。
    - `damo/speech_fsmn_vad_zh-cn-16k-common-pytorch` 。
    - `damo/speech_campplus_sv_zh-cn_16k-common` 。
    - `damo/punc_ct-transformer_zh-cn-common-vocab471067-pytorch` 。

## 5. 用户交互流程

1.  **环境准备**：用户首次运行启动脚本，Skill自动检测硬件、下载模型及FFmpeg。
2.  **任务提交**：用户拖拽或选择待处理的音频/视频文件。
3.  **参数配置（可选）**：用户可调整批量大小（如分段时长）、开启说话人分离、指定热词文件。
4.  **处理执行**：
    - 系统将视频解封装提取音频。
    - 音频经VAD分割后送入Paraformer进行解码。
    - CAM++模型对音频片段进行聚类，生成说话人标签。
5.  **结果输出**：同时生成带发言人标记的SRT/JSON/markdown 三种文件格式，返回文件地址。

## 6. 附加说明
- **并发处理**：针对单任务多文件场景，需引入线程锁（threading.Lock）处理模型并发推理时的缓存冲突问题，或建议用户使用串行队列以避免索引越界错误 。
- **错误恢复**：针对因音频损坏导致的解码错误，应跳过该片段并记录错误日志，而非终止整个长音频的转写任务。
